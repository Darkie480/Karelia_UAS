Kurssin päiväkirja: HackMD blogi

Blogi on ollut äärimmäisen työllistävä täyttää, koska oppiminen vie suuren osan jaksamisesta. Opittujen asioiden kunnollinen reflektonti jälkikäteen on ollut työlästä, raskasta ja turhauttavaa...

Blogin löydät joko täältä tai navigointipalkin linkistä.

Asetusten pitäisi olla niin, että luenta on sallittu kaikilta linkin haltijoilta.

  • Mitä tein?
    • Blogin, jossa käsittelen oppimaani opin edetessä.
  • Miten tein?
    • Käyttäen kurssilla toivottua HackMD-sivustoa.
  • Miksi teit?
    • Kuului kurssiin.
  • Mitä opin?
    • Blogin kirjoittaminen auttaa keskittämään oppia tiettyyn aihealueeseen.
  • Mitä jäi vielä oppimatta?
    • Muistiinpanojen kirjoittaminen on minulle vaikeaa ja vaatii lukematonta määrää luetun toistamista että luetusta tekstisä saa leipäosuuden kiinni.
  • Miten mielestäni onnistuin tässä tehtävässä ja mitä tekisin nyt toisin (jos mitään)?
    • Lähinnä haluaisin oppia kirjoittamaan muistiinpanoja ja ajatuksia rakentavammin. Olen enemmän tarinankertoja kuin kirjanpitäjä.

Tehtävä 1: HTML / CSS harjoittelu

Tehtävän ideana oli tutustua yleiseen web-ohjelmoinnin tyyliin sekä sivuston tyylittelyyn yksinkertaisella CSS ja HTML -harjoitteella. Ikäväkyllä kurssilla vaadittu Prettier-lisäosa VSCodessa on ollut turhauttava, koska HTML5 maailmassa suuri osa end-tageista on poistunut. W3Schools Validator on tämän vuoksi tarjonnut suuria määriä "INFO" herjauksia, joka on aiheuttanut turhaa siivoamistyötä jotta koodista saadaan siistiä.

Yleisesti HTML/CSS harjoitukset ovat itselleni mielekkäitä, koska olen harrastellut asian parissa valtaosan elämästäni, ylläpitäen yhä yksinkertaisia harrastesivuja osoitteessa https://www.darkies.fi.

Tehtävä 1: HTML / CSS harjoittelu

Värimaailma sekä sisältö on itse luotua, opettajan antamaa pohjakoodia en liiemmin muuttanut.

  • Mitä tein?
    • Pienen esittelysivun käyttäen HTML:ää ja CSS:ää. Pohjakoodi ja rakenne on sama kuin opettajan esimerkkikoodissa, mutta tein pieniä muutoksia sivuun saadakseni siitä enemmän itseni näköisen.
  • Miten tein?
    • Yksinkertaisia muutoksia molempiin tiedostoihin, harjoitus oli itselleni helppo aiemman harrastelun pohjalta.
      • Käytetyt kielet: HTML, CSS
      • Käytetyt ohjelmat: VS Code, Google Chrome
  • Miksi teit?
    • Tehtävänannon mukaisesti.
  • Mitä opin?
    • Tässä harjoituksessa uutta oppia tuli lähinnä usean DIV-elementin käytöstä liki toisiaan. En myöskään ollut käyttänyt onnistuneesti footereita aiemmin ja tämän harjoituksen jälkeen lisäsin ne myös harrastesivulleni.
  • Mitä jäi vielä oppimatta?
    • Aivan varmasti lukuisia asioita - HTML ja CSS maailma on suunnattoman suuri.
  • Miten mielestäni onnistuin tässä tehtävässä ja mitä tekisin nyt toisin (jos mitään)?
    • Kaikenkaikkeaan olen tyytyväinen sivustoon. En kokenut tarvetta suuremmalle hifistelymäärälle.

Tehtävä 2: JavaScript 1

Tehtävän ideana oli luoda JavaScriptillä lyhyt koodi, joka tarkistaa onko syötetty sana palindromi vai ei. Harjoitus tehtiin konsolipohjaisena, kirjoittaen ja suorittaen Visual Studio Codessa.

Tehtävä 2: JavaScript 1 harjoitus

Yksinkertainen harjoitus, joka kävi läpi sanan kirjain kirjaimelta, tarkistaen onko sana sama molempiin suuntiin.

En saanut koodia toimimaan välilyönnin kanssa. En tiedä miten olisi pitänyt toimia jotta toiminta olisi luonnistunut.

  • Mitä tein?
    • Simppeli palindromitarkastaja, joka palauttaa "true" arvolla konsoliin lausekkeen "Sana on palindromi" sekä "false" arvolla "Sana ei ole palindromi" (ks. kuva 1).
  • Miten tein?
    • Ohjelma on suoritettu JavaScriptillä VSCodessa. Koodissa alustetaan yksinkertaiset muuttujat ja luodaan simppeli ehtolause syötteen tarkistajalle. Erillinen funktio tarkistaa kirjain kirjaimelta jos syötetty sana tosissaan on palindromi (ks. kuva 2).
      • Käytetyt kielet: JavaScript
      • Käytetyt ohjelmat: VS Code, powershell
  • Miksi teit?
    • Pyrin toimimaan tehtävänannon mukaisesti.
  • Mitä opin?
    • Opin JavaScript koodirakennetta ja koodin alustamista. Koodirakenne on hyvin samanlainen kuin C#:ssa, mutta eriää kuitenkin kriittisissä paikoissa.
  • Mitä jäi vielä oppimatta?
    • Varmasti lukuaisia asioita. Olen yhä hyvin tuore koodien kanssa, eikä koodin luominen tyhjästä ole itselleni helppoa millään tavalla.
  • Miten mielestäni onnistuin tässä tehtävässä ja mitä tekisin nyt toisin (jos mitään)?
    • Olisin halunnut saada onnistumaan palindromitarkastuksen myös välilyönnillisillä sanoilla, kuten innostunut sonni, mutta olen tyytyväinen tulokseen.

Tehtävä 3: JavaScript 2

Tehtävänä oli luoda pieni konsolipohjainen puhelinluettelo, jossa voit etsiä henkilön numeroa sekä lisätä uuden henkilön parin valmiiksi alustetun henkilön seasta. Tehtävän ideana oli palauttaa haettu numero funktiolla.

Tehtävä 3: JavaScript harjoitus 2

En saanut tässä tehtävässä scandeja toimimaan, vaikka ne toimivatkin edellisessä JavaScript-harjoituksessa ongelmitta.

Alkuperäinen palautukseni oli lievästi väärä, koska funktioni palautti sekä numeron että tulosteen, opettajan huomautuksesta korjasin koodin tehtävänannon mukaiseksi.

  • Mitä tein?
    • Yksinkertainen JavaScript koodi, joka pyörii Switch-rakenteessa käyttäjän haluaman toiminnon mukaisesti. Ohjelma tarjoaa käyttäjälleen mahdollisuuden syöttää uuden henkilön, hakea numeroa, tulostaa koko puhelinluettelon tai lopettaa ohjelman. Ohjelma pyörii niin kauan kunnes käyttäjä valitsee 0 - lopeta.
  • Miten tein?
    • Tein switch-rakenteen, joka ehdottaa käyttäjälle toimintoa, jonka mukaan koodi hakee joko esisyötettyjä henkilöitä, tarjoaa mahdollisuutta lisätä listaan uuden henkiön (etunimi, sukunimi sekä puhelin), tulostaa koko luettelon tai poistua ohjelmasta.
      • Käytetyt kielet: JavaScript
      • Käytetyt ohjelmat: VS Code, powershell
  • Miksi teit?
    • Loop-rakenteissa Switch on alusta alkaen ollut itselleni mieluisin. Muutoin pyrin vain toteuttamaan tehtävänannon mukaisesti tehtävän.
  • Mitä opin?
    • JavaScript funktioita. Funktioiden yksinkertaistaminen sekä toiminnallinen siivoaminen on ollut itselleni haastavaa.
  • Mitä jäi vielä oppimatta?
    • On vielä paljon opittavaa eritoten tehokkaasta koodista. Refaktorointitapani ovat kovasti kehitystä vailla.
  • Miten mielestäni onnistuin tässä tehtävässä ja mitä tekisin nyt toisin (jos mitään)?
    • Yleisesti olen kovin tyytyväinen tehtyyn työhön, siihen en tosin koskaan löytänyt vastausta miksi ääkköset eivät tulostu konsoliin oikein. (ks. kuva 1)

Tehtävä 4: jQuery

Tehtävänä oli tehdä käyttöliittymä ennakkoon annettuun puhelinluetteloon, jossa oli vain taulukko henkilöistä. Tehtävään kuului myös ennalta annettu script, joka etsi puhelinluettelon henkilöt annetun netisivun kautta.

Tehtävä 4: jQuery harjoitus

Kursorin ollessa painikkeen päällä, painike muuttuu vihreäksi.

  • Mitä tein?
    • Tein yksinkertaisen, simppelin ja suhteellisen raikkaan värisen puhelinluettelon HTML:ää ja CSS:ää käyttäen. Koodin body-osuuteen on liitetty opettajan ennalta tarjoama script-pätkä muodostamaan lista scriptin sisällä asetetun linkin mukaisesti (ks. kuva 2). Tein myös hover-ominaisuuden painikkeen päälle, muuttaen sen vihreäksi.
  • Miten tein?
    • Yksinkertaista HTML ja CSS muokkausta ja pienen trial & error kokeilun jälkeen sain myös scriptin toimimaan oikealla tavalla.
      • Käytetyt kielet: JavaScript, HTML, CSS
      • Käytetyt ohjelmat: VS Code, Google Chrome
  • Miksi teit?
    • Tehtävänannon mukaisesti.
  • Mitä opin?
    • Opin perusperiaatteita jQueryyn liittyen. Opin myös että jQuery on siirtymässä syrjään eikä ole niin yleisesti käytössä enää.
  • Mitä jäi vielä oppimatta?
    • jQuery on konseptina vain pintaraapaistu, eli suurilta osin kokonaisuus jäi pimentoon. Ymmärrän periaatteen ja käyttötarkoituksen.
  • Miten mielestäni onnistuin tässä tehtävässä ja mitä tekisin nyt toisin (jos mitään)?
    • Olisin halunnut tehdä ominaisuuden, joka estää hakupainikkeen aktivoitumisen ellei nimi-kenttä ole täynnä.

Tehtävä 5: HTTP-metodi PUT

HTTP-metodeihin kurssilla tutustuttiin pienimuotoisen puhelinluettelon muokkaamisella. Tehtävänannossa oli erillinen client ja server, ideana lisätä "muokkaa" painike ominaisuuksineen ennalta annettuun taulukkoon, josta muokattu puhelinnumero tallennettiin serverille käyttäen fetch-mekaniikkaa, käyttäen http-metodia PUT.

Ennen tehtävänantoa harjoittelin erinäisiä bootstrap-ominaisuuksia, koska vaikka tunnistin tyylilajin, en ollut koskaan aiemmin käyttänyt tätä. Tästä kirjoitin oman tekstinsä blogiin viikon 9 kohdalle. Tässä kyseisessä harjoitteessa pyrin jättämään bootstrapin vähemmälle, koska HTTP-metodien parissa työskentely oli itselleni raskasta ja työlästä.

Tehtävä 5: HTTP-metodi PUT harjoitus

Värikäs ja selkeä ilmiasu oli itselleni mielekkäämpi työskennellä kuin harmaa-valkoinen. Eri elementtien visuaalinen muuttaminen antaa erilaista perspektiiviä

Kullakin painikkeella oli rooliin suunniteltu hover-väri, tuoden selkeyttä painikkeen toimintaan. Muokkaus-painike sekä muokkausten tallennuspainike keltainen, indikoiden näiden yhtenäisyyttä, punainen on geneerinen väri tuhoamiselle ja vihreä geneerinen väri uuden luonnille.

Rivinmuokkauksen pohjakoodin sai nopeasti mallinnettua aiemmista riveistä.

  • Mitä tein?
    • Tein puhelinluettelolle käyttöliittymän, johon lisäsin "muokkaa"-painikkeen, hover-värit kaikille painikkeille auttamaan erottamaan painikkeet toimintoineen toisistaan; uuden yhteystiedon luomisen napin väri muuttui vihreäksi, muokkaukseen liittyvät painikkeet keltaiseksi ja poistoon liittyvät painikkeet punaiseksi. (ks. kuva 2)
    • Tein myös toiminnallisuuden painikkeelle ennalta annetun script2.js-tiedoston pohjalta. (ks. kuvat 3 ja 4)
  • Miten tein?
    • Tutustuin Bootstrappiin elementteineen, suoritin tarvittavia muutoksia niin .html -tiedostoon kuin .js -tiedostoon ja muokkasin ulkoasua mieluisaksi itselleni käyttäen CSS -tyylitiedostoa.
      • Käytetyt kielet: HTML, CSS, JavaScript, JSON
      • Käytetyt ohjelmat: VS Code, Powershell, Notepad++
    • Tehtävä vaati myös terminaalissa pienen tietokannan käynnistämistä.
    • Koin suuria ongelmia koodipuolen kanssa, jonka vuoksi tapeltuani kokonaisen päivän scriptien kanssa päätin muokata visuaalista puolta itselleni mielekkäämmäksi. Ulkoasun erinäisten elementtien kanssa toimiessa tulee katsottua koodia eri tavalla, mahdollistaen paremman ymmärryksen. Tämä on auttanut itseäni lukuisat kerrat.
  • Miksi teit?
    • Yksinkertaisuudessaan koska tehtävänanto niin vaati. Tein visuaalista puolta todennäköisesti tarpeettoman paljon, koska tämä on tapa joka auttaa minua tekemään asioita luontevammin.
  • Mitä opin?
    • Oppia tuli niin JavaScriptiä, Bootstrappiä kuin HTTP-metodeista. Olen toiminut useita kertoja bootstrap-pohjaisten sivujen kanssa, mutta en halunnut luoda semmoista itse. HTTP-metodit ovat itselleni teorian asteella tuttuja, mutta käytännön harjoitukset tuottavat yhä haasteita. En ole täysin varma miksi tai miten sain koodin (kuva 4) toimimaan, mutta siinä kohtaa kun toiminta oli tapahtunut, lopetin koodin käsittelyn.
  • Mitä jäi vielä oppimatta?
    • Uskomaton määrä asioita koko tehtävän parissa. Kaikki tehtävän osa-alueet ovat niin laajoja käsitteitä, että oppi ei varmasti koskaan ole oikeasti valmista. W3Schoolsin sivuillekin jäi vielä lukemattomia välilehtiä ja otsikoita käsittelemättä, sekä lukuisia harjoituksia tekemättä. Myös serveripuoli jäi kovin vieraaksi.
  • Miten mielestäni onnistuin tässä tehtävässä ja mitä tekisin nyt toisin (jos mitään)?
    • Jos olisin ehtinyt ja jaksanut, olisin halunnut tehdä virheenkäsittelyn sivustolle, joka estää tyhjän kentän muokkauksen tai puhelinnumeron poistamisen muokkaamisen yhteydessä. Sivusto on kovin jatkokehitettävissä jaksamisen siihen antaessa myöten.

Tehtävä 6: Portfolio

Kurssiportfolion rakentaminen niputtaa ja viimeistelee ensimmäisen web-ohjelmoinnin kurssin tämän kyseisen sivun muodossa. Tehtävänantona oli käyttää sivuston luontiin bootstrap-elementtejä ja tyylittelyä.

Tehtävä 6: Portfolion rakentaminen

Sivu on yksinkertainen, värimaailmaltaan tekijänsä mieleinen.

Kuhunkin tehtävään on lisätty modaali, joka näyttää aiempien tehtävien ulkoasua muistelun vuoksi.

HackMD blogi on upotettu sivuun, josta sen voi lukea helposti. Linkki vie myös tarvittaessa HackMD sivulle, jossa opettaja voi tarjota kommentteja.

Siisteyden ylläpitämisen vuoksi kuvat on kaikki yhdessä kansiossa, tuotu header-fontti omassaan. Sikäli siisteyttä olisi voinut viedä pidemmälle ja laittaa alikansioihin vielä tehtävän mukaisesti.

Runsaiden modaalien vuoksi koodin luettavuus rupesi kärsimään tehtävän edetessä, yksinkertainen kommentti kunkin osion alkuun auttoi kovasti lunttaamaan missä kohdassa mennään, mitä pitäisi seuraavaksi tehdä.

  • Mitä tein?
    • Tein elämäni ensimmäisen portfolion. Portfolioon nivoutuu Karelian Ammattikorkekoulun Web-ohjelmointi 1 kurssin työt ja niiden läpikäynti. Tehtävänanto korosti asioita joihin tulisi vastata, tyylittelyä mitä pitäisi suosia ja kehote tehdä mahdollisimman yksinkertainen sivusto.
  • Miten tein?
    • Tein simppelin ja tummansävyisen HTML-sivuston, käyttäen Bootstrap 5.3 tyylittylä. Osan tyylilajista ohitin oman CSS-tiedoston kautta tuodakseni sivustolle enemmän omaa ilmettä ja uniikkia lookkia.
      • Käytetyt kielet: HTML, CSS
      • Käytetyt ohjelmat: VS Code, Paint, Snipping tool, Notepad++, Google Chrome
  • Miksi teit?
    • Tehtävänannon mukaisesti mutta mahdollisimman pitkälti oman persoonani mukaisesti.
  • Mitä opin?
    • Opin simppeleitä tapoja muokata bootstrap elementtejä ja yksinkertaisen joskin jonkin verran sekavan tavan luoda nettisivuja. Tämä on ensimmäinen sivusto jolle olen sisällyttänyt modaaleita ja erinäisiä painikkeita yksinkertaisten tekstilinkkien sijaan.
  • Mitä jäi vielä oppimatta?
    • HTML maailma ei ole koskaan valmis. Myös bootstrapin maailma on laaja ja monipuolinen, enkä saanut tämänkään harjoituksen myötä raapaistua pintaa syvemmälle. Harjoitus ei sisältänyt lainkaan JSONia eikä JavaScritiä, joten näiden osalta harjoitus jäi täydellisesti vajaaksi vielä.
  • Miten mielestäni onnistuin tässä tehtävässä ja mitä tekisin nyt toisin (jos mitään)?
    • Yleisesti olen tyytyväinen kokonaisuuteen, mutta jos tekisin uudelleen, haluaisin ehkä koittaa adaptoida sivun toimivaksi myös mobiililla. Kokeilin oman kotisivupalvelimeni kautta toimivuutta kännykällä, eikä elementit sopineet toisiinsa laisinkaan.
